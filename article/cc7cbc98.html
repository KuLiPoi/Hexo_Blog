为Hexo添加哔哔组件和主页滚动消息

发表于2021-06-03|更新于2021-06-03|笔记

这个是Hexo系列的第四篇了，
本来是想写点别的的，最近灵感有点枯竭了。
所以 我们还是继续折腾Hexo吧。
本次我们将为你的Hexo增加哔哔组件，本次安装将在Butterfly主题下进行操作，其他主题大同小异。

首页哔哔小组件

哔哔动态详情页

腾讯云相关内容配置

环境配置

  1. 注册并登录[腾讯云开发CloudBase](小程序云开发特惠-小程序云开发首购活动- 腾讯云 (tencent.com))
  2. 进入控制台 ，按需新建环境，
    选择空模板，上海地区，按量计费，环境名称随意，开启免费，立即开通
    免费资源有限，当免费资源耗尽届时会产生费用，如日访问量大于10000，请考虑付费套餐
  3. 打开登录授权页面，打开 “匿名登录”
  4. 打开安全配置页面，添加网站域名到 “安全域名”
  5. 打开数据库页面，新建数据库，命名为talks，权限为 用户可读管理员可写

函数部署 - bber

  1. 新建云函数
    打开新建云函数页面，新建函数名为 “bber”，其余保持默认并创建，进入函数代码编辑页面，复制代码内容覆盖至 index.js，修改预设的ServerKey，保存修改。

  2. 新建依赖包

    依旧在函数代码页面，新建文件 “package.json”，写入代码，点击保存并安装依赖。

  3. 配置HTTP访问

    打开HTTP访问服务页面，新建触发路径，触发路径默认为 /bber ，并关联你创建好的云函数即 bber。

  4. 待连接生成完毕后，打卡你的连接，即``https://你的连接/bber`，如果显示Key不匹配，即配置成功。

函数部署 - bb-talk

  1. 新建云函数并命名为 bber-talk，将代码覆盖至 index.js。
  2. 新建 package.json ，写入代码，保存并安装依赖。

函数部署 - bb-list

  1. 新建云函数并命名为 bber-list，将代码覆盖至 index.js。
  2. 新建 package.json ，写入代码，保存并安装依赖。

获取API地址

  1. 浏览器直接访问 https://你的地址/bber?key=bber(云函数默认为bber)&from=消息来源&text=消息内容
  2. 打开云存储页面，复制 bber-list.json 的临时连接 （这个连接总要用到，建议保存起来。）

主题配置

我这里用的主题为 Butterfly ，其他主题大同小异。

新建PUG模板

在路径 themes/butterfly/layout/includes/page新建 bb.pug
并修改其中的头像链接和bber-list连接

添加判断

在路径 themes/butterfly/layout修改 page.pug

添加网页

在路径 Hexo/source·新建 bb 文件夹，在文件夹中新建 index.md

编辑导航栏

如果 Hexo 版本5.0.0以上，编辑 Hexo 根目录下的 _config.butterfly.yml
反之则编辑 themes/butterfly/_config.yml

添加首页滚动组件

在路径 themes/butterfly/layout/includes新建 bbTimeList.pug
并修改其中bber-list连接

将组件添加到主页

在路径 themes/butterfly/layout修改 index.pug

引入相关CSS

如果 Hexo 版本5.0.0以上，编辑 Hexo 根目录下的 _config.butterfly.yml
反之则编辑 themes/butterfly/_config.yml

检测成果

快使用Hexo三连 Hexo clean && hexo g && hexo s 测试看看吧
